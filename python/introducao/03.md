# Literais e Variáveis
Você está andando na rua e começa a olhar as coisas ao redor, daí você vê "Macarrão R$4,99", "Leve 3 por preço de 2" etc. etc. Você sabe que cada um dos elemento desse texto representa algo, como "Macarrão" que é uma palavra, 4,99 que seria um número com casas decimais e 3 que é um número inteiro; cada um representa coisas diferentes.

No mundo da programação, também temos as divisões desses símbolos, e chamamos de **tipo**! Cada tipo ocupa um espaço na memória, variando de linguagem para linguagem. Em Python, e em outras linguagens, temos uma peculariedade que será explicada na introdução à orientação objeto. Antes de tudo, vamos entender a diferença entre literais e variáveis.

## Literais
Seria realmente o conteúdo bruto, algo que "não ocupa memória". Como assim? No modo interativo do Python, caso se digite, entre aspas, "Olá, Mundo!", o interpretador vai compreender como um literal, pois não vai ser algo guardado na memória; assim como digitar 10, 19.87 (**os decimais são representados por ponto (.) e NÃO por vírgula (,)**), -8 e assim por diante.

## Variáveis
Vamos pensar numa grande caixa de papelão. Nela será colocado outras caixas menores dentro e cada uma delas terá alguma coisa coisa: uma vai ter pregos, outra parafusos, outras porcas, dentro outras coisas. A caixa de papelão gigante é toda a sua memória disponível para usar no seu programa, as caixinhas pequenas são às **variáveis** e **constantes** (espaços ocupados nessa memória) e os parafusos são o conteúdo delas.

Variáveis são espaços delimitados na memória que podem receber valores, e estes podem depois ser modificados (por isso o nome *variável*, algo que *varia*). As constantes são também um espaço na memória que podem receber valores, mas **NÃO** podem ser modificados ao longo do programa, podendo evitar possíveis erros de segurança.

### Prática
Para se declarar (criar) uma variável em Python é muito simples: `nome_da_variavel = valor`. O `=` indica atribuição, então o código anterior se lê "O `valor` é atribuído à variável `nome_da_variavel`", ou seja, a variável está recebendo o `valor`.

Em Python, por padrão, não se tem como declarar uma constante. O que se faz aqui é usar uma convesão e esperar que o desenvolvedor respeite-a. Essa conversão seria que uma constante teria seu nome em caixa alta (exemplo: `CONSTANTE = valor`). Logicamente, isso por dar futuros problemas, pois, por algum descuido, esse valor pode ser sobreescrito e o interpretador não avisará sobre. Por esse motivo que há maneiras de evitar isso, que será abordado mais para frente.

## Tipos
Toda variável, constante e litetal tem um tipo, abaixo listamos os principais:
### Tipos numéricos (int, float, complex)
* **int**: Representa o conjunto dos números inteiros (exemplos: 1, 2, -4, -1000);
* **float**: Representa o conjunto dos números reais, ou seja, aqueles que apresentam vírgulas, a diferença aqui que segue o padrão internacional que usa o ponto (exemplos: 1.4, -5.65412...., 1.0, -2.0);
* **complex**: Representa o conjunto dos números complexos, em que é divididos em duas partes: a *real* e a *imaginária*, que é cada uma tem seu ponto flutuante (o tipo **float**). Para acessar cada as duas partes de uma variável *num*, usa-se `num.real` e `num.imag` para acessar a parte real e a imaginária, respectivamente.

É possível transformar outros valores em numéricos usando as respectivas funções: `int(valor)`, `float(valor)` e `complex(valor)` (ou `complex(real, imag)`).

### Tipo Booleano (bool)
O tipo boollean (*boolean* do inglês) representa o Falso (*False*) e o Verdadeiro (*True*), e são representado por dois valores constantes: `True` e `False`.
O Boolean é uma subclasse (subtipo) dos inteiros, pois, pela máquina, é representado como 0 e 1.

Para converter um valor para booleano usa-se `bool()`.

Para o Python, todo valor `False` é definido como:
* `None` e `False`;
* Zero em qualquer valor numérico (exemplos: 0, 0.0, 0b0 - para representar binário, etc.);
* sequência ou coleções de valores vazias: `""`, `''`, `()`, `[]`, `{}`, `set()`, `range(0)`. [Não se preocupe, isso será explicado mais tarde]

O restante é considerado `True`.

### Texto (str ou String)
Os textos em Python são tratadas como `str` (ou *Strings*), e são representados como aquilo que está entre as aspas simples (''), duplas ("") ou triplas(""""""). Enquanto as aspas simples e duplas são para textos em uma linha só, as aspas tripls são para diversas linhas! Segue os exemplos abaixo:
```python
texto1 = "Ola"
texto2 = 'Oi'
texto3 = """
Desta maneira, o aumento do diálogo entre os diferentes setores produtivos facilita a criação dos níveis de motivação departamental.
"""

#Erro: por conta da aspa duplas não suportarem essa quebra de linha
texto4 = "
Discordo
" 

#Erro: por conta da aspa simples não suportarem essa quebra de linha
texto5 = '
Discorodo da sua discordância
'
```

Para se ter uma quebra de linha usando as aspas simples ou duplas, usa-se o `\n`, que, pela conversão internacional, significa a quebra de linha.

Você deve estar se perguntando: "Dá para usar aspas duplas dentro de aspas duplas?", e a resposta é que SIM! Mas com ressalvas... Quando se abre as as aspas (independente de qual seja) o texto "fecha" quando encontra de novo elas, por isso que as palavras ficam entre aspas. Vamos ver o exemplo abaixo:
```python
texto = "Aqui tem um texto" # aqui não tem nenhum erro, que as aspas abrem e fecham corretamente
texto_feio = "Aqui "tem uma citação" de Ronaldinho Gaúcho" # percebe que o texto já até muda de cor no Visual Studio (ou em qualquer outra IDE), isso indica que o texto não está entre as aspas, está fora delas!

# Muita vezes precisamos mostrar que há a presença de aspas no texto, e para termos isso usamos a barra invertida antes da nova aspas
texto_lindo1 = "Agora \"tem uma citacao\" do Ronaldinho Gaúcho" # não terá mais o errro e o código vai ficar com corzinha
texto_lindo2 = 'Agora \'tem uma citacao\' do Cristiano Ronaldo'
texto_lindo3 = """Agora \"""tem uma citacao\""" do Ronaldo Fernômeno"""
```

Agora quiser escrever as aspas simples dentro de aspas duplas, vice versa não tem nenhum problema!
```python
lero_lero1 = "'O sonho é a realização de um desejo que uma parte valiosa e bem dotada da sociedade já está implícito no uso contemporâneo da interpretação transferencial sem que isso impeça a extração de gozo.'"
lero_lero2 = '"Acima de tudo, é fundamental ressaltar que o entendimento das metas propostas representa uma abertura para a melhoria das posturas dos órgãos dirigentes com relação às suas atribuições."'
lero_lero3 = """
"A equipe de suporte precisa saber que o módulo de recursão paralela corrigiu o bug na compilação de templates literais."
"""
```

Uma pergunta que deve surgido é: "Por que usar barra invertida se é só trocar as aspas?". Bem, acaba mais por ser uma boa prática usar aspas simples para apresentar letras (exeplos: 'a', 'b',...), aspas duplas textos e frases longas (exeplo: "Uma frase") e a tripla para quando se tem um texto longo para ser formato na mão no código, como editar um texto para um email.

#### Concatenação em strings
A concatenação significa pegar uma string e junto com algum outro texto (ou valor). A concatenação varia de linguagem para linguagem, precisando ficar atento sobre.

Em Python acaba por ser um assunto delicado, que vai emendar com conteúdos mais para frente.

Há apenas uma maneira de concatenar textos que é usando o mais (`+`), e essa junção só pode acontecer com outros textos (não podendo ser com `int`, `float` ou `bool`). Por exemplo:
```python
texto1 = "Olá"
texto2 = "Mundo!"
textos_juntos = texto1+texto2
print(textos_juntos) # "OláMundo!" (sim, o "Olá" e o "Mundo!" vão estar colados)
print("Tudo"+"bem?") # "Tudobem?"
```

Mas, talvez veja em outros lugares a "concatenação" com vírgula (`,`). Na real, isso não é uma concatenação, isso é uma propriedade que ficará mais claro no conteúdo de funções.

O Python tem tipagem dinâmica, ou seja, a variável pode mudar de tipo mesmo depois de declarada:
```python
teste = "OI"
teste = 10
# não ocorrerá erro no código
``` 
Mas ele tem uma tipagem forte, ou seja, ele não vai fazer conversões automáticas de tipo para o código funcionar. Com isso, tentar concatenar um texto com um número dará erro:
```python
teste1 = "OI"
teste2 = 10
teste3 = teste1 + teste2 # erro, pois não é possível juntar um texto com um número
print(teste1 + teste2) # erro, pois não é possível juntar um texto com um número
``` 

Mas como podemos escrever um texto com as variáveis? Transformando ela em *String* usando `str()`:
```python
teste1 = "OI"
teste2 = 10
teste3 = teste1 + str(teste2) # OI10
print(teste1 + str(teste2)) # OI10
```

# Boas práticas (convesões que ajudam no entedimento do código por você e terceiros)
* **Nome de variáveis**: elas devem ser
    * *Sem acentos*: **NÃO** pode declarar qualquer coisa com acento na programação, por conta das linguagens serem escrita baseada na língua inglêsa e ela não tem acentos, então dará erro qualquer nome acentuado;
    * *Sem caracter especial*: O `/`,`\`,`:`,`;`,`-`,`@`, dentre outros caracters especiais **NÃO** são permitidos na hora de escrever o código, muitas fezes por conta de serem resenvadas para determinados propósitos ou por não serem reconhecidas pela linguagem como nome;
    * *Curtas*: nome de variáveis muito grande pode poluir o código ainda prejudir a leitura;
    * *Descritivas*: os nomes devem descrever o que aquela variável significa, como, por exemplo, uma que representa o peso de uma pessoa pode ser chamado de `peso`, para altura, `altura` e assim por diante;
    * *Em um padrão de escrita*: quando for passando nos fóruns de dúvidas, artigos e textos de programação, você verá diversas formas de escrever nomes de variáveis (e outras coisas da programação), e isso são padrões de escritas e temos diversas delas, entre eles:
        * **camelCase**: escrever as palavra do nome toda junta, em que cada uma inicia com letra maiúscula, com exceção da primeira. Exemplo: nomeDaVariavel;
        * **PascalCase**: semelhante ao camelCase, mas com a primeira também maíuscula, quase que obrigatório para delcarar classe (veremos mais para frente). Exemplo: NomeDaVariavel;
        * **snake_case**: as palavras são separadas por um *underline* e estão toda minúsculas, sendo o padrão na comunidade de Python. Exemplo: nome_da_variável;
        * **UPPER_SNAKE_CASE**: quando usa o padrão snake_case em caixa alta, quase obrigatório para as constantes. Exemplo: NOME_DA_VARIÁVEL;
        * **kebab-case**: Usa hífen (-) para separar palavras, usados principalmente para rotas/URL de sites, mas que na maioria das linguagens de programação não se aplica para dar nomes. Exemplo: rota-para-site.
