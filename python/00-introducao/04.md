## Operadores lógicos
Estes operadores são um tanto diferente dos aritméticos, pois eles vão verificar a "verdade" da expressão. Como assim? Pense na situação: 7 > 4? Sim, então isto é verdadeiro (`True`). 7 < 4? Não, então é falso (`False`).
\
Operadores lógicos, quando usados, irão retornar um valor Verdadeiro (`True`) ou Falso (`False`).
\
Com isso, temos os operadores de comparação e os operadores booleanos.

Irá compara dois valores, mas de maneira booleana. Como assim? Veja a tabela abaixo:

<p>Notas:</p>
<ol>
    <li>No geral, <code>or</code> vai verificar se pelo menos uma é verdadeira. Se <code>x</code> for verdadeiro, a expressão toda é; se y é verdeiro, a mesma coisa. Todavia, se ambos for falso (nem <code>x</code>, nem <code>y</code> é verdadeiro), a expressão é falsa. Outro ponto é que, se o primeiro valor (<code>x</code>) já for verdadeiro, não se verifica o <code>y</code> e já retorna <code>True</code>;</li>
    <li>No geral, o <code>and</code> verifica se as duas é verdadeiras. Se o <code>x</code> ou o <code>y</code> (ou ambos) forem falso, a expressão retorna <code>False</code>. Caso contrário, retorna <code>True</code>. Caso o primeiro valor da expressão (<code>x</code>) já for falso, já é retornado o valor <code>False</code>;</li>
    <li>No geral, <code>not</code> retorna o valor lógico oposto de <code>x</code>. Caso x seja <code>True</code>, retorna <code>False</code>; caso contrário, retorna <code>True</code>.</li>
</ol>

Todo o valor (texto, número etc.) tem um valor lógico, em que todo valor `False` é definido como:
* `None` e `False`;
* Zero em qualquer valor numérico (exemplos: 0, 0.0, 0b0 - para representar binário, etc.);
* sequência ou coleções de valores vazias: `""`, `''`, `()`, `[]`, `{}`, `set()`, `range(0)`. [Não se preocupe, isso será explicado mais tarde]

O restante é considerado `True`.
\
Então, usando `True` e `False`, podemos atribuir valores à `x` e `y`.

**Exemplos:**
```python
x = True
y = False
print(x and y) # False
print(x or y) # True

x = False
y = False
print(x and y) # False
print(x or y) # False

x = True
y = True
print(x and y) # True
print(x or y) # True

x = False
print(not x) # True

x = True
print(not x) # False
```

Em Python temos algo mais interessante. Analisando o que cara [operação](#operadores-booleano) realmente faz, certas "jogadas" de códigos que podemos fazer:
\
**Exemplos**

```python
x = 0
y = 10

print(y or x) # ou print(y or x)
# Printará o y (10), pois o OU leva em consideração os valores considerados verdadeiros
# com isso, printará 10.

print(y and x) # ou print(x and y)
# Contrário do OU, AND considera os falso, imprimindo então o 0

## Mas o que acontece se os valores são todos positivos?
x = 30
y = 10

print(y or x) # o OR acaba por considera o primeiro verdadeiro, então será imprimido o y
print(x or y) # aqui o x
print(x and y) # O AND considera o valor falso como predominante, por não ter mais, acaba por considerar o primeiro, ou seja, o x
print(y and x) # aqui será o y

## A lógica para os valores falsos são iguais para o positivos, aplicando as lógicas do OR e AND
x = 0
y = [] # não se preocupem para saber o que é isso

print(y or x) # 0
print(x or y) # []
print(x and y) # 0
print(y and x) # []
```
